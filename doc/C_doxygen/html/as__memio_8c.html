Function: as_memio_open

This function allocates memory for each HW module buffer and initializes the as_memio infrastructure for buffer management. Depending on settings of "memio_config" and "flags" the setup for the corresponding hardware module is performed. If a NULL-pointer is provided for "memio_config", the default values are used. The "flag" paramter determines whether a as_memreader or as_memwriter is associated. Supported flags are O_RDONLY (as_memwriter) and O_WRONLY (as_memreader). If the former is used, only calls to "read" are allowed with the "write" interface becoming unavailable and vice versa for O_WRONLY. The function returns a pointer to the opaque as_memio_file_s structure which can be used for performing further actions on the as_memio module.

Parameters
configThis structure is used to setup the buffer(s) and hardware modules (see "as_memio_config_init")
flagsThis variable is used to determine the mode in which to use the ASTERICS streaming interface. Depending on this variable either the "as_memreader" or "as_memwriter" hardware module is used for the streaming interface.
Returns
The function returns a pointer to the configured structure which can be used for further calls.

Function: as_memio_read

The purpose of this function is to request data from the as_memwriter hardware module and copies the requested amount but not more than the currently available amount of data into "buffer". This function returns the number of bytes which were actually transfered.

Parameters
fdPointer to the opaque as_memio structure.
bufferPointer to the buffer for copying the data from hardware to.
countRequest a desired amount of data from the hardware module in byte. This value has to be a multiple of the configured bus interface width of the as_memwriter.
Returns
The function returns the number of bytes which were actually transfered to "buffer".

const void *  buffer,
uint32_t  count

Function: as_memio_write

The purpose of this function is to transfer data from the user software to the buffer which is used by the as_memreader hardware module. This function may copy less data from the user buffer than requested by "count" if the buffer has not enough empty entries.

Parameters
fdPointer to the opaque as_memio structure.
bufferThe container which holds data meant for transferring to hardware.
countSpecifies the amount of data requested by the caller of this function in byte. This value has to be a multiple of the configured bus interface width of the as_memreader.
Returns
This function returns the number of bytes which were actually copied from "buffer".

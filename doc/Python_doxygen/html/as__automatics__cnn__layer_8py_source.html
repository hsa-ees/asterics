144  """! @brief Configure this CNN layer accelerator module.

147  @note This method must be used BEFORE the layer is connected!
148  @param operation:
149  Operation this layer should implement.
150  Supported: ["CONV2D", "CONV2D_LARGE", "POOL"]
151  Default: "CONV2D"
152  @param input_bit_width:
153  Bit width of the data to process.
154  Valid values: [1 .. n]; Default is [8]
155  @param output_bit_width:
156  Bit width of the output of a single filter module.
157  Total output width is determined by output_bit_width * filter_count
158  Valid values: [1 .. n]; Default is [8]
159  @param kernel_size:
160  Size of the quadratic convolution kernel.
161  Informs the number of required weight values per kernel.
162  Valid values: >= 3 ; Must be odd numbers; Default is [3]
163  @param input_channel_count:
164  Number of input channels.
165  A seperate set of kernel values is expected per input channel.
166  Each filter performs a multi-channel convolution over all input
167  channels and outputs the accumulated and quantized result.
168  Valid values: [1 .. n]; integers; Default is [1]
169  @param filter_count:
170  The number of filters that are to be implemented.
171  Default is [1]
172  @param strides:
173  Reduces the size of the output image.
174  1 = no reduction, 2 = keep every second column and row,
175  3 = keep every third column and row, ...
176  Default is [1]
177  @param activation_function:
178  Define the activation function of the filter modules.
179  Valid values depend on the configured operation:
180  For "CONV2D": ["relu", "none"]; Default is "none"
181  For "POOL": ["max", "none"]; Default is "none"
182  @param weight_values:
183  Python list or Numpy array containing the weight values to use.
184  The weights must be 8 bit signed integers.
185  The values are exptected to be in the following order: 'NHWC'
186  [output channels/filters, value rows, value columns, input channels]
187  Can also be provided via a numpy file using 'weights_npy_file'.
188  @param bias_values:
189  Python list or Numpy array containing the bias values to use.
190  The biases must be 32 bit signed integers.
191  The values are expected in the order of the output channels.
192  Can also be provided via a numpy file using 'biases_npy_file'.
193  @param quantization_factors:
194  Python list or Numpy array containing the factors to use.
195  The values must be 32 bit floating point numbers.
196  The values are expected in the same order as the biases.
197  These values are used to scale the filter responses back to int8.
198  Can also be provided via a numpy file using
199  'quantization_factors_npy_file'.
200  @param weights_npy_file:
201  Path and file name of the numpy file containing the weight values.
202  See 'weight_values' for more information.
203  @param biases_npy_file:
204  Path and file name of the numpy file containing the bias values.
205  See 'bias_values' for more information.
206  @param quantization_factors_npy_file:
207  Path and file name of the numpy file containing the quantiation factors.
208  See 'quantization_factors' for more information.
209  @param filters_per_module:
210  Number of filters to implement within a serial convolution module.
211  If = 1: Implement all filters in parallel: Uses most amount of resources
212  with greatest processing speed. Note that this is only possible
213  for !very! small networks
214  If > 1: Implement multiple filters per module in serial processing:
215  Lowers resource usage and linearly reduces processing speed.
216  Default is [1]: fully parallel operation
217  @param quantization_offset_value:
218  8 bit integer value applied to the outputs of this layer.
219  Used by tensorflow lite to achieve an offset number representation.

221  @param weight_accuracy:

224  implemented in hardware as fixed mathematical bit shifts.
225  By reducing the number of factors allowed per weight, hardware
226  resources can be saved.
227  Valid values are [1 .. 4].
228  Default is [4] = full accuracy.

97 PLL_LockLostDetect: process(PixelClkIn)
98 begin
99  if (pRst = '1') then
100  pLocked_q <= (others => '0');
101  pLockWasLost <= '1';
102  pLockGained <= '0';
103  elsif Rising_Edge(PixelClkIn) then
104  pLocked_q <= pLocked_q(pLocked_q'high-1 downto 0) & pLocked;
105  pLockWasLost <= (not pLocked_q(0) or not pLocked_q(1)) and pLocked_q(2); --two-pulse
106  pLockGained <= (pLocked_q(0) and not pLocked_q(1));

110 -- The TMDS Clk channel carries a character-rate frequency reference
111 -- In a single Clk period a whole character (10 bits) is transmitted
112 -- on each data channel. For deserialization of data channel a faster,
113 -- serial clock needs to be generated. In 7-series architecture an
114 -- OSERDESE2 primitive doing a 10:1 deserialization in DDR mode needs
115 -- a fast 5x clock and a slow 1x clock. These two clocks are generated
116 -- below with an MMCME2_ADV/PLLE2_ADV.
117 -- Caveats:
118 -- 1. The primitive uses a multiply-by-5 and divide-by-1 to generate
119 -- a 5x fast clock.
120 -- While changes in the frequency of the TMDS Clk are tracked by the
121 -- MMCM, for some TMDS Clk frequencies the datasheet specs for the VCO
122 -- frequency limits are not met. In other words, there is no single
123 -- set of MMCM multiply and divide values that can work for the whole
124 -- range of resolutions and pixel clock frequencies.
125 -- For example: MMCM_FVCOMIN = 600 MHz
126 -- MMCM_FVCOMAX = 1200 MHz for Artix-7 -1 speed grade
127 -- while FVCO = FIN * MULT_F
128 -- The TMDS Clk for 720p resolution in 74.25 MHz
129 -- FVCO = 74.25 * 10 = 742.5 MHz, which is between FVCOMIN and FVCOMAX
130 -- However, the TMDS Clk for 1080p resolution in 148.5 MHz
131 -- FVCO = 148.5 * 10 = 1480 MHZ, which is above FVCOMAX
132 -- In the latter case, MULT_F = 5, DIVIDE_F = 5, DIVIDE = 1 would result
133 -- in a correct VCO frequency, while still generating 5x and 1x clocks
134 -- 2. The MMCM+BUFIO+BUFR combination results in the highest possible
135 -- frequencies. PLLE2_ADV could work only with BUFGs, which limits
136 -- the maximum achievable frequency. The reason is that only the MMCM
137 -- has dedicated route to BUFIO.
138 -- If a PLLE2_ADV with BUFGs are used a second CLKOUTx can be used to
139 -- generate the 1x clock.

89  -- Slave register configuration:
90  -- Allows for "dynamic" configuration of slave registers
91  -- Possible values and what they mean:
92  -- "00": Register "off". Register will not be available and won't be implemented -> No hardware resource utilization.
93  -- "01": From HW view: Status register. Data transport from hardware to software. HW can only write, SW can only read.
94  -- "10": From HW view: Control register. Data transport from software to hardware. HW can only read, SW can only write.
95  -- "11": Combined Read/Write register. Data transport in both directions.
96  -- When both sides attempt to write simultaniously, only the HW gets to write.
97  -- These registers use both the slv_ctrl_reg and slv_status_reg ports for communication.
98  constant slave_register_configuration : slv_reg_config_table(0 to 6) := (AS_REG_STATUS, AS_REG_STATUS, AS_REG_STATUS,
99  AS_REG_CONTROL, AS_REG_STATUS, AS_REG_STATUS, AS_REG_STATUS);
